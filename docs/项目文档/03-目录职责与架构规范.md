## 前端 Monorepo 分层架构

### 架构原则

项目采用 **Internal Packages 模式** —— 所有共享包以原始 `.ts`/`.tsx` 文件导出（无预编译），由消费方的 bundler 直接编译。这实现了零配置 HMR 和即时跳转定义。

### 依赖方向（严格单向）

```
views/  →  core/ + ui/
                ↕
          core 与 ui 相互独立
```

**硬约束：**

- `packages/core/` — 零 react-dom、零 localStorage（用 StorageAdapter）、零 process.env、零 UI 库
- `packages/ui/` — 零 `@multica/core` 导入（纯 UI，无业务逻辑）
- `packages/views/` — 零 `next/*` 导入、零 `react-router-dom` 导入、零 Store 直接使用
- `apps/web/platform/` — 唯一允许使用 Next.js API 的地方
- `apps/desktop/.../platform/` — 唯一允许使用 react-router-dom 的地方

### 各包详细职责

#### `packages/core/` — 无头业务逻辑

所有共享 Zustand Store 驻留于此，即使是视图相关的 Store（筛选、视图模式）—— Store 是纯状态，不是 UI。

### 后端分层架构

```
┌─────────────────────────────────────────┐
│             HTTP Handler                │  ← Chi Router
│    (请求解析、验证、响应格式化)            │
├─────────────────────────────────────────┤
│            Service 层                    │
│    TaskService, AutopilotService,       │
│    EmailService                          │
├─────────────────────────────────────────┤
│           sqlc 生成层                    │  ← 类型安全的 SQL
│    (pkg/db/generated/)                   │
├─────────────────────────────────────────┤
│          PostgreSQL 17                   │
└─────────────────────────────────────────┘

   ┌─────────────────────────────────────┐
   │         事件驱动层（旁路）            │
   │                                     │
   │  Event Bus → Listeners:             │
   │    ├→ WS Hub (广播到客户端)          │
   │    ├→ Subscriber (自动订阅)          │
   │    ├→ Activity Log (审计记录)        │
   │    ├→ Notification (收件箱)          │
   │    └→ Autopilot Sync (状态同步)      │
   └─────────────────────────────────────┘
```
