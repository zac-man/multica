# 数据库模型与后端事件系统

## 核心数据库表

## Issue 多态指派人

Issue 的 `assignee_type` + `assignee_id` 实现了多态指派：

```
┌─────────────────┐     ┌─────────────────┐
│     Member       │     │      Agent      │
│  (人类成员)      │     │   (AI Agent)    │
└────────┬────────┘     └────────┬────────┘
         │                       │
         │ assignee_type='member'│ assignee_type='agent'
         │                       │
         └───────────┬───────────┘
                     │
              ┌──────┴──────┐
              │    Issue     │
              │  assignee_id │
              └─────────────┘
```

**同理适用于**：`creator_type/creator_id`、`author_type/author_id`、`actor_type/actor_id`、`recipient_type/recipient_id`。

## 后端事件系统

### 事件总线架构

```mermaid
flowchart LR
    A[Handler 写入 DB] --> B[发布事件到 Bus]
    B --> C{类型分发}
    C --> D[类型特定 Handler]
    C --> E[全局 Handler (SubscribeAll)]
    D --> F[WS 广播 Listener]
    D --> G[Subscriber Listener]
    D --> H[Activity Listener]
    D --> I[Notification Listener]
    D --> J[Autopilot Listener]
```

### 事件监听器职责

| 监听器 | 文件 | 职责 |
|--------|------|------|
| WS 广播 | `listeners.go` | 将事件广播到工作空间 WS Room（个人事件用 SendToUser） |
| 自动订阅 | `subscriber_listeners.go` | 创建者、指派人、评论者、@被提及者自动订阅 Issue |
| 活动日志 | `activity_listeners.go` | 记录 Issue/Comment/Agent 操作到审计日志 |
| 通知 | `notification_listeners.go` | 指派、提及、状态变更等创建收件箱通知 |
| Autopilot 同步 | `autopilot_listeners.go` | 关联 Issue 终态或 run_only 完成时同步 Run 状态 |

### WebSocket Hub 房间模型

```
┌─────────────────────────────────────────┐
│                  Hub                     │
│                                         │
│  Room: ws_abc123                        │
│  ┌─────┐ ┌─────┐ ┌─────┐              │
│  │User1│ │User2│ │User3│              │
│  └─────┘ └─────┘ └─────┘              │
│                                         │
│  Room: ws_def456                        │
│  ┌─────┐ ┌─────┐                       │
│  │User1│ │User4│   ← User1 可同时在     │
│  └─────┘ └─────┘      多个工作空间      │
│                                         │
│  BroadcastToWorkspace(ws_abc123, msg)   │
│  SendToUser(User1, msg)                 │
│  Broadcast(msg)                          │
└─────────────────────────────────────────┘
```

**连接认证流程**：
1. Web 端：HttpOnly Cookie JWT（WS 握手自动携带）
2. Desktop/CLI 端：首条消息 `{"type":"auth","payload":{"token":"..."}}`

**心跳机制**：
- 服务端每 54 秒发 Ping（90% × 60s pong 等待）
- 未响应 Pong → 连接断开
- 慢客户端（发送缓冲区满）→ 连接断开
